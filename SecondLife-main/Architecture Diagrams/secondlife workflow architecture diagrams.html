Boot sequence: On startup a background thread calls init_db() (seeds 25 patients + 3 hospitals), then pipeline.load() (reads all CSVs into memory), then pipeline.train() (builds RF model or loads model_cache.pkl). The Flask server is immediately available during boot; GET /api/status returns ready: false until the thread completes.

2. Database Schema

SQLite database (secondlife.db) with 5 tables. JSON columns (conditions, medications, documents, research_conditions) are stored as TEXT and parsed by _row_to_dict() in database.py.

erDiagram patient_accounts { TEXT id PK TEXT username UK TEXT password_hash TEXT synthea_id "null for hand-made patients" TEXT first_name TEXT last_name TEXT dob TEXT gender TEXT address TEXT conditions "JSON array of strings" TEXT medications "JSON array of strings" TEXT documents "JSON array of file metadata dicts" INTEGER open_to_trials "0 or 1" TEXT created_at } hospital_accounts { TEXT id PK TEXT username UK TEXT password_hash TEXT hospital_name TEXT location "City, ST format" TEXT research_conditions "JSON array of strings" TEXT created_at } patient_trial_interests { TEXT id PK TEXT patient_id FK TEXT trial_id "NCT number" TEXT trial_title REAL match_score TEXT status "interested or withdrawn" TEXT created_at } connections { TEXT id PK TEXT patient_id FK TEXT hospital_id FK TEXT trial_id "nullable NCT number" TEXT trial_title TEXT initiated_by "patient or hospital" TEXT status "pending accepted rejected completed" TEXT message TEXT created_at } connection_messages { TEXT id PK TEXT connection_id FK TEXT sender_role "patient or hospital" TEXT sender_id TEXT body "max 2000 chars" TEXT created_at INTEGER is_read "0 or 1" } patient_accounts ||--o{ patient_trial_interests : "saves interests" patient_accounts ||--o{ connections : "initiates or receives" hospital_accounts ||--o{ connections : "initiates or receives" connections ||--o{ connection_messages : "contains messages"
Uniqueness: connections has a composite unique index idx_conn_unique ON connections(patient_id, hospital_id, COALESCE(trial_id, '')) to prevent duplicate connection requests even when trial_id is NULL. patient_trial_interests has UNIQUE(patient_id, trial_id). connection_messages has index idx_msgs_conn ON (connection_id, created_at).

Cache behaviour: If model_cache.pkl exists on disk, train() loads the cached RF + calibrator and skips the ~2-3 min training step. Delete the file to force a full retrain (required after changing features or pipeline logic).

7. Feature Engineering — 17 Features

All features are normalised to 0–1. Computed by _compute_features(patient_row, trial_id) in pipeline.py. Used for both training pseudo-labels and inference scoring.
